# Система управления библиотекой (Семинар 1)

**Роль:** Все роли (Архитектор, Разработчик, Тестировщик, Технический писатель).

## 🛠 Инструкция по сборке и запуску
Проект настроен для работы в среде WSL2 (Ubuntu).

```bash
mkdir build && cd build
cmake ..
make
./LibrarySystem
```

## 📂 Структура проекта
- `include/` — заголовочные файлы (.h)
- `src/` — реализация классов (.cpp)
- `main.cpp` — точка входа и тестирование
- `CMakeLists.txt` — конфигурация сборки

## 📝 Отчет техписателя (Проблемы и решения)
В процессе работы возникли следующие технические моменты, решенные с помощью ИИ-консультанта:

**Проблема 1:** CMake не видел файлы в папке src.
- *Запрос:* "CMake Error at CMakeLists.txt: Cannot find source file: src/Book.cpp".
- *Решение:* Была исправлена структура папок и проверен регистр имен файлов. Пути в CMakeLists.txt приведены в соответствие с реальным расположением.

**Проблема 2:** Ошибка компиляции "undefined reference to main".
- *Запрос:* "Почему при сборке через make вылетает ошибка ld: undefined reference to main?"
- *Решение:* Выяснилось, что файл main.cpp находился не в корне, или не был включен в список исходников add_executable. Файл был перенесен в корень проекта.
